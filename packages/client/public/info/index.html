What it does

notapipe lets you move data from one device to another instantly.


  • ✓Open a session
  • ✓Join from another device (link or QR code)
  • ✓Paste, type, or drag and drop files
  • ✓Close the session and it disappears

Direct peer-to-peer transfer

Everything happens directly between devices. No cloud. No backend.


  • ✓No server stores or reads your content
  • ✓No account required
  • ✓Close the session and data is gone

Why it exists

Most tools turn everything into a file, message, or document. But sometimes you just need to:

  • ✓Move data between work and personal devices
  • ✓Transfer files without email or cloud storage
  • ✓Share something sensitive
  • ✓Use a shared or temporary machine
  • ✓Leave no trace

Sessions are ephemeral by default, but you can enable local storage or export your data at any time. There is no server-side database — your data only exists if you choose to keep it.

Installed as a PWA, notapipe can run independently of notapipe.app, and can be configured to use your own signaling and TURN infrastructure.

How it works

Step 1
Peer-to-peer connection

Devices connect directly over WebRTC. The session starts with a shared link or a QR code, then the peers establish a private connection between them.

Step 2
Encrypted transport

All traffic is end-to-end encrypted. Only the connected peers can read the data, and no server ever sees the shared content.

Step 3
Local-first sync

Text and files sync using local CRDT state over the encrypted channel. There is no server-side storage unless you explicitly enable persistence.


Connection Options

QR code (recommended for maximum privacy): The connection details are encoded in a QR code, so the devices can connect directly without any server involvement at all.

- or -

Signalling server: A signalling server is only used to establish the initial connection. You can configure or replace it, and once the peers are connected the data flows directly between devices.


No server ever sees your data.
